Class AhciController

Namespace
Cosmos.Kernel.HAL.Devices.Storage
Assembly
Cosmos.Kernel.HAL.dll

One initialized AHCI HBA. Each controller owns its ABAR (HHDM-mapped), capability snapshot, port list, and a per-controller command region allocated from PageAllocator — kernel-virtual for memset, physical (via VirtualToPhysical(ulong)) for the HBA's CLB / FB / CTBA registers. The driver scans PCI for every matching SATA/AHCI controller and instantiates one AhciController per match, so a system with two HBAs gets two instances and all of their ports show up in Ports.

Methods

Initialize()

Bring this controller up: enable bus master + memory decoding, map ABAR via HHDM, allocate a command region via PageAllocator, enable AHCI mode, snapshot CAP, and probe every implemented port. After a successful call, Ports contains one Sata per attached SATA drive. Returns false if any precondition fails — the caller (Initialize()) then drops this controller and moves on. Init-time failures are reported via return value rather than thrown so a misconfigured device can't take the boot down.
