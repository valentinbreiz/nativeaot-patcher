Classes

Ahci

AHCI driver entry point. Scans PCI for every MassStorageController / SataController device and brings each one up independently — a machine with two SATA HBAs gets two AhciController instances and all of their attached SATA drives show up in Ports.

AhciController

One initialized AHCI HBA. Each controller owns its ABAR (HHDM-mapped), capability snapshot, port list, and a per-controller command region allocated from PageAllocator — kernel-virtual for memset, physical (via VirtualToPhysical(ulong)) for the HBA's CLB / FB / CTBA registers. The driver scans PCI for every matching SATA/AHCI controller and instantiates one AhciController per match, so a system with two HBAs gets two instances and all of their ports show up in Ports.

NvmeController

One initialized NVMe controller. Owns its admin and a single IO queue pair, and the namespaces it discovered. The admin queue is polled (it only runs once at init); the I/O queue is interrupt-driven via MSI-X so callers of Read(uint, ulong, Span<byte>, ushort) / Write(uint, ulong, ReadOnlySpan<byte>, ushort) can yield instead of burning a CPU on a spinloop, and up to Cosmos.Kernel.HAL.Devices.Storage.NvmeController.IoQueueDepth-1 commands can be in flight concurrently because every slot owns its own 4 KiB DMA bounce buffer.

If MSI-X is unavailable (e.g. ARM64 today, or a pathological PCI device with no MSI-X capability) the controller falls back to polled completion, but in that mode commands are serialized through an internal mutex.

Limitations:

  • One IO submission and one IO completion queue (qid=1), depth 8. At most depth-1 commands are in flight (NVMe queue-full rule: a depth-N SQ holds N-1 entries, or the wrapped tail is indistinguishable from an empty queue).
  • Single-PRP transfers — every command moves at most one 4 KiB page through the slot's bounce buffer, so PRP2 is always 0. Namespaces whose LBA format exceeds one page (or carries metadata) are skipped at discovery.
NvmeIoOp

NVMe NVM-command-set IO opcodes (NVM Express 1.4 §6.x).

NvmeNamespace

One NVMe namespace exposed as an IBlockDevice. Reads and writes are issued one LBA at a time through the parent controller's per-slot bounce buffers — multiple namespaces (or concurrent callers on the same namespace) execute in parallel up to the controller's I/O queue depth minus one (the NVMe queue-full rule).

NvmeRegisters

NVMe controller register block accessor. Wraps the BAR0 virtual base (BAR0 phys + HHDM offset) and exposes the spec registers via Native.MMIO.

Layout follows NVM Express 1.4 §3.1.

PortRegisters

AHCI Port Registers. Carries a back-reference to the AhciController that owns the port so per-port code (SATA command issue, port reset) can reach controller state without going through globals.

Sata

SATA drive port driver. The bounce buffer is a single page allocated from PageAllocator; the HBA DMAs to its physical address while ReadDataBlock16(ushort[]) / WriteDataBlock8(ReadOnlySpan<byte>) touch the kernel-virtual address.

Error contract: command issue throws on failure (no free slot, device timeout, task-file error), so ReadBlock(ulong, ulong, Span<byte>) / WriteBlock(ulong, ulong, ReadOnlySpan<byte>) never silently hand back stale bounce-buffer contents. All I/O on a port is serialized by an internal lock — the port owns a single bounce buffer.
