Namespace Cosmos.Kernel.System.Filesystems.Fat

Classes

FatBootSector

Parsed BIOS Parameter Block plus derived geometry for a FAT volume. All *Lba fields are absolute on the underlying IBlockDevice the BPB was read from.

FatDirEntry

Parsed FAT directory entry. ByteOffset is the byte position of the 8.3 record within the buffer that produced it; LFN entries that contributed to Name precede it.

FatDirectory

Parser and encoder for FAT directory clusters. Operates over raw byte spans; no I/O is performed here.

FatFilesystemType

FAT12 / FAT16 / FAT32 driver entry point. Pluggable into the VFS via VfsManager.RegisterFilesystem. Source strings:

  • Empty: use the injected device passed to the constructor (test seam).
  • "<index>": partition index in Partitions.
FatFormatOptions

Parameters for FatFilesystemType.TryFormat(System.ReadOnlySpan<char>, IVfsFormatOptions?). All 0 / null fields mean "let the formatter pick a sensible default for the requested FatType and the device size."

FatTable

FAT12 / FAT16 / FAT32 table accessor and chain manager. All entry reads and writes go through the underlying IBlockDevice using FatBootSector geometry. Cluster numbers coming from on-disk metadata are untrusted: accessors treat anything outside the volume's data clusters as end-of-chain and never let it drive I/O.
