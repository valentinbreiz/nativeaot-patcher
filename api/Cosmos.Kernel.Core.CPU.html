Namespace Cosmos.Kernel.Core.CPU

Classes

InternalCpu

Low-level CPU operations that can be used by Core components like the heap. Native imports live in Bridge/Import/CpuNative.cs.

InterruptManager

Architecture-neutral interrupt registration and routing. Owns the dense 256-entry handler table and the dynamic-vector allocator used by MSI programmers. The actual dispatch path (ack/EOI/LPI/fatal-fault handling) is delegated to the platform IInterruptController implementation in Cosmos.Kernel.Core.X64 / Cosmos.Kernel.Core.ARM64.

MsiRouting

Architecture-neutral PCI MSI / MSI-X binding. Both x64 (LAPIC Fixed-mode messages) and ARM64 (GICv3 ITS GITS_TRANSLATER writes) ultimately program the same shape into a device's MSI-X table — a 64-bit address plus a 32-bit data dword — but the routing model differs:

  • x64: data = IDT vector chosen by the kernel; address encodes the destination LAPIC ID. No per-device prep.
  • ARM64 ITS: data = EventID (per-device index); address = the shared GITS_TRANSLATER doorbell. The ITS must be told the (DeviceID, EventID) → LPI mapping out of band before the device fires.

This file abstracts both behind IMsiBinder: each platform initializer registers a binder, then HAL-level PCI MSI-X code calls PrepareDevice(uint, uint, uint, int) once per device and BindEntry(object?, int, IrqDelegate, uint, out ulong, out uint) per MSI-X table entry. The binder owns vector / LPI allocation and any device-specific bookkeeping (ITT allocation, MAPD, MAPTI on ARM64).

PCI device identity is passed in raw (bus, slot, function) form to keep this file free of HAL/Pci dependencies — Core can't reference HAL upstream.

IRQContext
InternalCpu.InterruptScope

A disposable scope that disables interrupts on creation and restores the prior interrupt-enable state on dispose. Critically, this saves the full RFLAGS / DAIF on entry and restores it on exit, so nested scopes are correct: an inner scope disposing inside an outer disabled region does NOT prematurely re-enable interrupts.

ICpuOps

Essential CPU operations for multi-architecture support

IInterruptController

Platform-specific interrupt controller. Implementations live in Cosmos.Kernel.Core.X64 (APIC/IDT) and Cosmos.Kernel.Core.ARM64 (GIC/Exception Vectors). Each owns the arch-specific dispatch path: EOI timing, ack semantics, fatal-fault handling, and any extra handler tables (e.g. GICv3 LPIs) all live in the implementation.

IMsiBinder

Platform-specific MSI binding backend. Implemented once per arch.
